# CG4j Benchmark Scripts

Standalone Python scripts for comparing call graphs generated by CG4j.

## Prerequisites

- Python 3.7 or higher
- pip (Python package manager)

## Setup

Install dependencies:

```bash
pip install -r benchmark/requirements.txt
```

## Usage

Compare two call graph CSV files:

```bash
python benchmark/compare_cg.py <cg1.csv> <cg2.csv>
```

**Example:**

```bash
python benchmark/compare_cg.py cg_okhttp_w_deps_wala.csv cg_okhttp_w_deps_asm.csv
```

**Output:**

```
Call Graph Comparison
======================================================================================
+----------+---------------------------+--------------------------+------------------+
| Metric   | CG1                       | CG2                      | Difference       |
+==========+===========================+==========================+==================+
| File     | cg_okhttp_w_deps_wala.csv | cg_okhttp_w_deps_asm.csv | -                |
+----------+---------------------------+--------------------------+------------------+
| Nodes    | 4,468                     | 3,874                    | -594 (-13.3%)    |
+----------+---------------------------+--------------------------+------------------+
| Edges    | 13,319                    | 25,432                   | +12,113 (+90.9%) |
+----------+---------------------------+--------------------------+------------------+
```

## Understanding the Metrics

### Nodes
- **Definition**: Unique methods that appear in the call graph (either as caller or callee)
- **Calculation**: Union of all methods in `source_method` and `target_method` columns
- **Example**: If method `A` calls `B`, and `B` calls `C`, there are 3 nodes: `A`, `B`, `C`

### Edges
- **Definition**: Call relationships between methods
- **Calculation**: Total number of rows in the CSV (excluding header)
- **Example**: If method `A` calls `B` twice (different call sites), there are 2 edges

### Difference
- **Format**: `+/-X (+/-Y%)`
- **Interpretation**:
  - Positive (+): CG2 has more nodes/edges than CG1
  - Negative (-): CG2 has fewer nodes/edges than CG1
  - Percentage shows relative change: `(CG2 - CG1) / CG1 * 100`

## CSV Format

Expected format (default CG4j output):

```csv
source_method,target_method
package/Class.method:(descriptor),package/Class.method:(descriptor)
```
