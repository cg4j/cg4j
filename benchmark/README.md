# CG4j Benchmark Scripts

This folder contains standalone Python scripts for benchmarking and comparing call graphs generated by CG4j.

## Overview

The benchmark scripts allow you to:
- Compare two call graph CSV files side-by-side
- View statistics on nodes (unique methods) and edges (call relationships)
- Calculate differences and percentage changes between call graphs

## Prerequisites

- Python 3.7 or higher (available via project's `.venv`)
- pip (Python package manager)

## Setup

### 1. Activate the Project's Virtual Environment

From the **project root directory**:

```bash
source .venv/bin/activate
```

### 2. Install Benchmark Dependencies

```bash
pip install -r benchmark/requirements.txt
```

This installs:
- `tabulate` - For pretty-printing comparison tables

## Usage

### Basic Comparison

Compare two call graph CSV files:

```bash
# From the project root
python benchmark/compare_cg.py <path-to-cg1.csv> <path-to-cg2.csv>
```

### Examples

**Example 1: Compare two call graphs in the project root**

```bash
python benchmark/compare_cg.py cg_okhttp_w_deps_wala.csv cg_okhttp_w_deps_asm.csv
```

**Example 2: Compare call graphs from the cgs/ directory**

```bash
python benchmark/compare_cg.py cgs/slf4j_cg.csv cgs/slf4j_cg_with_rt.csv
```

**Example 3: Using absolute paths**

```bash
python benchmark/compare_cg.py /tmp/output1.csv /tmp/output2.csv
```

**Example 4: From within the benchmark/ directory**

```bash
cd benchmark
python compare_cg.py ../cg_okhttp_w_deps_wala.csv ../cg_okhttp_w_deps_asm.csv
```

### Expected Output

The script displays a formatted table showing:
- File names
- Node counts (unique methods in the call graph)
- Edge counts (total call relationships)
- Differences (absolute and percentage)

## CSV Format Requirements

The benchmark scripts expect call graph CSV files in the following format:

```csv
source_method,target_method
package/Class.method:(descriptor),package/Class.method:(descriptor)
...
```

This is the default output format from CG4j. See the main [README.md](../README.md) for more details.

## Understanding the Metrics

### Nodes
- **Definition**: Unique methods that appear in the call graph (either as caller or callee)
- **Calculation**: Union of all methods in `source_method` and `target_method` columns
- **Example**: If method `A` calls `B`, and `B` calls `C`, there are 3 nodes: `A`, `B`, `C`

### Edges
- **Definition**: Call relationships between methods
- **Calculation**: Total number of rows in the CSV (excluding header)
- **Example**: If method `A` calls `B` twice (different call sites), there are 2 edges

### Difference
- **Format**: `+/-X (+/-Y%)`
- **Interpretation**:
  - Positive (+): CG2 has more nodes/edges than CG1
  - Negative (-): CG2 has fewer nodes/edges than CG1
  - Percentage shows relative change: `(CG2 - CG1) / CG1 * 100`

## Troubleshooting

### Error: "tabulate library not found"

**Solution**: Install dependencies:
```bash
source .venv/bin/activate
pip install -r benchmark/requirements.txt
```

### Error: "File not found"

**Cause**: The CSV file path is incorrect or the file doesn't exist

**Solution**: 
- Verify the file path is correct
- Use absolute paths or paths relative to where you're running the script
- Check that the file exists: `ls -l <path-to-csv>`

### Error: "Invalid CSV format"

**Cause**: The CSV file doesn't have the expected headers

**Solution**:
- Ensure the CSV was generated by CG4j
- Verify the first line contains: `source_method,target_method`
- Check the file isn't corrupted: `head -5 <path-to-csv>`

### Virtual Environment Not Activated

**Symptom**: `python` or `pip` commands fail or use system Python

**Solution**: Always activate the virtual environment first:
```bash
source .venv/bin/activate
```

Verify activation:
```bash
which python  # Should show: /path/to/java-cg-wala/.venv/bin/python
```

### Permission Denied

**Cause**: Script isn't executable (shouldn't happen, but just in case)

**Solution**:
```bash
chmod +x benchmark/compare_cg.py
```

## Deactivating Virtual Environment

When you're done using the benchmark scripts:

```bash
deactivate
```
